Introduction

Documentation workspace for Smart Agriculture and Smart Water Management System architecture, sensors, devices, and operations.

Introduction

This wiki page provides a structured documentation system for CDAC SmartSystems. It is designed for engineering teams, operators, administrators, and students working with Smart Agriculture and Smart Water Management deployments.

Smart Agriculture: field sensors, irrigation control, soil telemetry, gateway networking, and farm dashboards.
Smart Water: flow monitoring, pressure telemetry, tank levels, pump control, SCADA/HMI links, and water distribution analytics.

Architecture

System Overview

The architecture should document how sensors, edge gateways, local services, dashboards, and backend databases communicate across the deployment. Use the placeholders below for IP addresses, network protocols, VLANs, and device roles.

Edge Layer: sensors, PLCs, microcontrollers, and local controllers.
Gateway Layer: MQTT bridge, Modbus gateway, LoRaWAN gateway, or Wi-Fi router.
Application Layer: Flask service, dashboards, CTF module, admin panel, and APIs.
Storage Layer: SQLite for local CTF/user data and optional time-series storage for telemetry.

# Operational Manual

## 1. System Startup
1. Power on [RASPBERRY_PI_OR_SERVER].
2. Verify network IP: [SERVER_IP_ADDRESS].
3. Start application:
   ```bash
   source .venv/bin/activate
   waitress-serve --host=0.0.0.0 --port=[PORT] app:app
   ```
4. Open dashboard: [DASHBOARD_URL].

## 2. Health Checks
| Check | Command / URL | Expected Result |
|---|---|---|
| Web App | http://[SERVER_IP_ADDRESS]:[PORT] | Page loads |
| Admin Panel | http://[SERVER_IP_ADDRESS]:[PORT]/admin.html | Login page loads |
| Database | [DB_PATH] | SQLite file exists |
| Sensor Gateway | ping [GATEWAY_IP_ADDRESS] | Reply received |

## 3. Daily Operations
- Check active sensors: [SENSOR_STATUS_URL]
- Verify agriculture telemetry: [AGRI_DASHBOARD_URL]
- Verify water telemetry: [WATER_DASHBOARD_URL]
- Export backup: [BACKUP_COMMAND]

## 4. Backup Procedure
```bash
mkdir -p backups
cp data/cdac_ctf.sqlite3 backups/cdac_ctf_$(date +%F_%H-%M).sqlite3
```

## 5. Restart Procedure
```bash
pkill -f waitress-serve
waitress-serve --host=0.0.0.0 --port=[PORT] app:app
```
